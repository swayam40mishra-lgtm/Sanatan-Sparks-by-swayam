1. Concept Recap

PocketTrip = AI + Wikipedia + Travel Planner. It helps budget and curious travelers discover destinations, local stories, and build a useful itinerary (complete with packing tips and safety notes).

  • Audience: Budget travelers, students, solo explorers.
  • Core idea: Enter trip parameters → receive a story-driven itinerary, packing checklist, and local cultural insights.
  • Tagline: Plan Smart. Travel Soulfully.

2. App Design — UI/UX + Color Palette

Color Palette

Primary colors and usage guide (hex):

  • Primary: #2B8CFF — Sky Blue (buttons, primary accents)
  • Secondary: #FFD166 — Golden Sand (cards, small accents)
  • Background: #F7FAFC — Off-white (page background)
  • Text: #1A202C — Dark Charcoal
  • Accent: #06D6A0 — Mint Green (success, positive actions)
  • Error: #EF476F — Coral Red (warnings)

Typography & Style

  • Font: Poppins or Inter.
  • Corner radius: 16px for cards and components.
  • Card style: Soft drop shadow, subtle gradients (sky → mint on hero).
  • Icons: Use Lucide or Feather for a minimal travel feel.

UI/UX Principles

  1. Keep the trip flow short: Input → Generate → Refine → Save.
  2. Show human context (local story) near itinerary days.
  3. Provide quick actions for bookings and sharing.
  4. Mobile-first: large tappable targets, compact timeline view.

3. Page-by-Page Structure

Home Page

Purpose: Capture trip intent quickly and sell the idea with a clean hero CTA.

Sections

  • Navbar: Logo, Discover, Plan, About.
  • Hero: Headline, small illustration, trip form (From, To, Dates, Budget, Interests), CTA: Plan My Trip.
  • 3 Feature Cards: Smart Itineraries, Local Stories, AI Packing Hacks.

Trip Results Page

When user clicks Plan, present an interactive itinerary and action cards.

Layout

  • Left: Day-by-day timeline (expandable day cards).
  • Right: Summary cards: Hotels, Trains, Packing Tips, Local Folk Story, Safety Notes.

Card Behavior

  • Hotel cards show price, rating, and quick-book CTA.
  • Packing tips show local weather-sensitive items.
  • Local story shows 150–300 words with source / citation.

Discover Page

Search and explore places and stories. Grid cards, filter by theme (history, food, spiritual circuits).

Profile Page

  • Saved Trips
  • Favorite Stories
  • Preferences (budget, food)
  • Badges & Achievements

4. Backend — How it works (simplified)

Three core parts

  1. Data Storage: JSON files or Firebase (users.json, trips.json, stories.json)
  2. Core Logic: Local simulated generator → stores trips.json and returns structured JSON for UI
  3. Output Rendering: Frontend reads trip JSON and builds cards/timeline

Example trips.json structure

{
  "trips": [
    {
      "id": "t101",
      "user": "Swayam",
      "from": "Lucknow",
      "to": "Jaipur",
      "budget": 5000,
      "dates": {"from":"2025-11-10","to":"2025-11-13"},
      "itinerary": [
        {"day":1, "activities": ["Bazaar", "Heritage walk"]},
        {"day":2, "activities": ["Temple", "Riverfront lunch"]}
      ],
      "hotels": [{"name":"Hotel Comfort","price":1200,"stars":3}],
      "createdAt": "2025-11-01"
    }
  ]
}

Flow (simple)

  1. User fills form → frontend gathers inputs
  2. Frontend calls generator (local function or later OpenAI)
  3. Generator returns JSON → frontend stores it and renders results
